== Description ==

世界に光を与えるといわれる女神オーロラ。
その名を冠した **Aurora Heatmap** は美しいヒートマップでユーザー行動を視覚化。
あなたのウェブサイトの活性化・最適化に光をもたらします。

= サイト運営で一番大切なこと =

それは *ユーザーが満足しているのか* ということ。

* コンテンツのどこを見てユーザーが動くか？
* ユーザーが迷っていないか？

Aurora Heatmap は、それを可視化する **最強のツール** です。

1. 導線の有効性
2. コンバージョン率
3. 見込み客、読者の離脱
4. Google の評価

悩みの改善点が見えてきます。

= プラグインの特徴 =

***No Coding***
***No Setting***

あなたが行うのはプラグインをインストールして有効化するだけ。
面倒なユーザー登録やセットアップは必要ありません。
ほとんどの WordPress 環境でデフォルトのままで動きます。
そして、Aurora Heatmap は **完全にプラグインだけで完結** します。

無料版は PC とモバイルそれぞれのクリックヒートマップを確認でき、何サイトでも利用いただけます。
無料でも PV や解析ページ数による制限はありません。

= 特記事項 =

キャッシュ系プラグインとの併用でうまく動作しない場合、JavaScript 関連の最適化をオフにするか、jQuery と Aurora Heatmap の計測スクリプト (reporter.js) を最適化の対象から外してください。
詳しくは、[公式サイトの解説ページ](https://market.seous.info/ja/aurora-heatmap#oc-1) をご覧ください。

以下のプラグインと併用できます。

* WP Rocket
* W3 Total Cache
* WP Super Cache

= 使い方とサポート =

より詳細な使い方や FAQ などの情報は [Aurora Heatmap 公式サイト](https://market.seous.info/ja/aurora-heatmap) で提供しています。
それらのドキュメントであなたの疑問の答えを見つけられない場合、WordPress.org の [サポートフォーラム](https://wordpress.org/support/plugin/aurora-heatmap/) をご利用ください。
なお、有料版には優先メールサポートがあります。

= プライバシーについて =

このプラグインは、以下の動作を **行いません** 。

* ユーザーの追跡
* 外部サーバーへの記録データ送信
* クッキーの利用
* IP アドレスを含む個人を特定できるデータの記録

== Installation ==

1. WordPress 管理者にログインします。
2. [プラグイン] タブをクリックします。
3. [新規追加] をクリックして、「Aurora Heatmap」を検索します。
4. [今すぐインストール] をクリックし、[プラグインを有効化] をクリックします。
5. 分析が開始され、しばらくすると訪問者の行動を見ることができます。

1.2.0 より、無料版・有料版ともに翻訳データを WordPress.org の GlotPress に一本化しました。
日本語表記に切り替わらない場合、ダッシュボードの「WordPress の更新」より翻訳の更新を行ってください。

== Frequently asked questions ==

= ヒートマップ使用にあたり、アカウントの登録や契約は必要ですか？ =

Aurora Heatmap は多くのヒートマップサービスとは違い、完全にお使いの WordPress の中で動作します。
アカウントの登録や契約は一切必要ありません。
ただし、有料版のご利用には、月間 または 年間のライセンス契約が必要です。

= 解析できるページ数や PV に制限はありますか？ =

有料版はもちろん、無料版においても、制限は一切設けていません。

= PC からモバイルのヒートマップを閲覧しても、PC 用のページが表示されてしまう =

レスポンシブデザインではなく User-Agent で表示を切り替えている場合は、モバイルからヒートマップを閲覧してください。

= 精度設定はどちらを使うべきですか？ =

おすすめは、正確なヒートマップのために、記録するデータを選別している *高精度* です。
記録が不十分な場合は、緩い選別の *標準* をお試しください。

精度設定の変更には、以下の点にご注意ください。

1. 標準から高精度への変更 …… これまでの標準データは消去されます。
2. 高精度から標準への変更 …… その時点から標準データの記録が始まります。

= プラグインをインストール後、どのくらいでヒートマップが見られますか？ =

サイトへのアクセス数にもよりますが、数時間～2日ほどでデータが蓄積されてきます。

= ヒートマップはリアルタイムに表示されますか？ =

ユーザーの行動は、必要に応じて一定頻度で Ajax 通信を経由して記録されます。
記録された情報は、再読込で画面に反映されます。

= 3日ほど経過しましたが、データがないようです。 =

アクセスが極端に少ないサイトでは、満足に記録されない可能性があります。
サイトに流入があることを確認してください。
キャッシュ系のプラグインを使用している場合は、Aurora Heatmap の計測スクリプトが出力されていない可能性があります。一度、キャッシュを削除してください。

= 自分でアクセスしても、記録されません。 =

データの信頼性を確保するため、特定のアルゴリズムで記録するデータを選別しています。
また、管理者のアクセスは記録から除外しています。

= いろんなページにアクセスがあるはずなのですが、トップページしかカウントされません。 =

デフォルトパーマリンク（`?p=123` 形式）の場合は、パラメータ形式の URL となります。
設定の「URL パラメータ」が統合の場合、全てトップページとみなされてしまいますので、別表示に設定ください。

= アクセス数と比較してヒートマップに記録される件数が少ないように見えます。 =

ヒートマップは絶対数のカウントではなく、ユーザー行動の傾向の把握が目的となります。
データの信頼性を高めるために、特定のアルゴリズムで記録するデータを選別しています。
記録される件数は、精度設定にも影響されます。

= パーマリンクが変更になった場合、ヒートマップデータは引き継げますか？ =

現在の仕様では、ヒートマップのデータを URL と結びつけて保存しているため、パーマリンクや投稿のスラッグなどを変更して URL を変更した場合、別ページとして扱われます。

= スマートフォンでヒートマップを閲覧できますか？ =

1.0.2 よりモバイル端末でのヒートマップ閲覧に対応しました。

= カウントバーとは何ですか？ =

縦軸のどの位置でいくつのデータが記録されているかを示します。
行動解析の助けになるでしょう。

= 記録されたデータは、外部に漏洩しないのでしょうか？ =

無料版・有料版ともに、外部サーバへのヒートマップデータの送信は一切ありません。
